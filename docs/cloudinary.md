## 🛠 Technology Stack & Mechanics

Our process relies on the following packages on the Node.js backend:
- [`cloudinary`](https://www.npmjs.com/package/cloudinary) (^2.9.0): The official Cloudinary SDK for uploading and managing assets.
- [`multer`](https://www.npmjs.com/package/multer): A middleware for handling `multipart/form-data`, primarily used for uploading files.
- [`streamifier`](https://www.npmjs.com/package/streamifier): Converts the Buffer generated by multer into a readable stream that can be piped directly to Cloudinary.

Multer is configured to use memory storage, ensuring files are kept in the server's RAM as Buffers instead of writing them to the local disk:

```javascript
const upload = multer({ storage: multer.memoryStorage(), limits: { fileSize: 500 * 1024 * 1024 } });
```

### 2. Upload Utility (`backend/utils/cloudinaryUtils.js`)
We abstract the upload logic into a utility function `uploadToCloudinary`. It takes a file Buffer and uses `streamifier` to pipe it to Cloudinary's `upload_stream`:

```javascript
const { v2: cloudinary } = require('cloudinary');
const streamifier = require('streamifier');

function uploadToCloudinary(buffer, options) {
    return new Promise((resolve, reject) => {
        const stream = cloudinary.uploader.upload_stream(options, (err, result) => {
            if (err) return reject(err);
            resolve(result);
        });
        streamifier.createReadStream(buffer).pipe(stream);
    });
}
```

The utility also exposes a `detectType` function to categorize files (e.g., separating images, videos, PDFs, and presentations based on the MIME type and file extension) which is helpful when deciding how to serve or parse them later.

### 3. Route Handlers (e.g., `backend/routes/admin.js`, `backend/routes/users.js`)
When an endpoint needs to handle a file upload, it uses the `multer` middleware first. For instance:
```javascript
router.post('/avatar', upload.single('avatar'), async (req, res) => {
```

Then the file buffer is passed to our utility:
```javascript
const result = await uploadToCloudinary(req.file.buffer, { folder: 'users/avatars' });
// result.secure_url contains the public URL of the uploaded image
// result.public_id contains the unique identifier for deletion/management
```

If we need to delete an old resource before uploading a new one, we do so via:
```javascript
await cloudinary.uploader.destroy(old_public_id, { resource_type: 'image' }).catch(() => {});
```

### Summary
1. Client uploads file as `multipart/form-data`.
2. Backend (Express) intercepts it with `multer` and holds the file in Memory (`req.file.buffer`).
3. Backend converts the buffer to a stream via `streamifier` and pipes it to Cloudinary.
4. Cloudinary responds with the file's new CDN URL and `public_id`.
5. We save the URL and `public_id` in our SQLite database.
